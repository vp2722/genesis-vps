# Incident Response Runbook

## Severity Definitions
| Severity | Description | Response Time |
|----------|-------------|---------------|
| SEV1 | Complete outage | Immediate |
| SEV2 | Major degradation | 15 minutes |
| SEV3 | Minor degradation | 1 hour |
| SEV4 | Minimal impact | 4 hours |

## Response Procedure

### 1. Detection
- Automated alerting
- User reports
- Monitoring dashboards

### 2. Triage
- Assess severity
- Identify impact
- Page appropriate responders

### 3. Communication
- Update status page
- Notify stakeholders
- Regular updates

### 4. Mitigation
- Implement temporary fix
- Reduce impact
- Restore service

### 5. Resolution
- Implement permanent fix
- Verify resolution
- Document timeline

### 6. Postmortem
- Schedule within 48 hours
- Blameless analysis
- Action items

## Escalation Path
1. On-call engineer
2. Team lead
3. Engineering manager
4. CTO

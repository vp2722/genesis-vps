# Retrospective Command

Conduct a structured retrospective to capture learnings.

## Usage
```
/retrospective [sprint|project]
```

## Process
1. Gather what went well
2. Identify what could improve
3. Define action items
4. Document in retrospectives folder

## Template

```markdown
# Retrospective - [DATE]

## What Went Well
- {item1}
- {item2}

## What Could Improve
- {item1}
- {item2}

## Action Items
| Action | Owner | Due |
|--------|-------|-----|
| {action} | {agent} | {date} |

## Key Metrics
- Velocity: {points}
- Bugs: {count}
- Tech debt addressed: {hours}

## Learnings to Capture
- [x] Added to learning system
```

Save to `.claude/learning/retrospectives/`.
